# LocationCacheLab 

Лабораторна робота з розробки Android-додатків. Проєкт реалізує функціонал отримання поточної геолокації користувача, її відображення на екрані та офлайн-кешування отриманих координат у локальну базу даних.

## Технологічний стек
- **Мова програмування:** Kotlin (версія 2.1)
- **UI Фреймворк:** Jetpack Compose (Modern UI)
- **База даних (Офлайн-кеш):** Room DB + KSP (для генерації коду)
- **Сервіси:** Google Play Services Location (для отримання точних координат)
- **Архітектура:** MVVM (Model-View-ViewModel) + Clean Architecture principles
- **Платформа/Java:** Android SDK 35 / Java 21

## Опис архітектури проєкту (MVVM)
Додаток побудований за класичною архітектурною схемою **MVVM**, що забезпечує розділення логіки та інтерфейсу:
1. **Model (Data Layer):** - `Room Database` — локальне сховище, яке зберігає історію локацій навіть без доступу до інтернету.
   - `LocationRepository` — відповідає за координацію між базою даних та сервісом геолокації Google.
2. **ViewModel (UI Logic Layer):**
   - Обробляє життєвий цикл екрана, запускає фонові запити на отримання локації та підписується на зміну даних (Flow) з бази даних Room. Передає в UI вже готовий стан (State).
3. **View (UI Layer):**
   - Реалізований на `Jetpack Compose`. Чистий декларативний інтерфейс, який просто відображає поточні координати, кнопку оновлення та список збережених у кеші локацій.
